# JavaScript Track (학습 기록)

이 폴더는 JavaScript를 **이론(문서화) + 실습(인터랙티브)** 두 갈래로 학습한 내용을 기록합니다.  
목표는 “개념을 이해하는 것”에서 끝나지 않고, **직접 만들며 체득하는 것**입니다.

---

## 📌 폴더 구성

### 1) `js-beginner/` (이론 + 학습 + 문서)

- JavaScript 핵심 개념을 **정리/설명**하고,
- 예제 코드로 “왜 그렇게 동작하는지”를 이해합니다.
- 목적: **개념을 언어로 설명 가능**한 상태 만들기

✅ 주로 다루는 주제

- 변수/타입, 스코프/호이스팅/TDZ
- 객체/배열, 함수, 클로저
- 프로토타입/클래스 기초
- 비동기 기초(Promise/async-await) 정리

📂 바로가기: [js-beginner](./js-beginner)

---

### 2) `js-core/` (학습 + 인터랙티브 만들기)

- 배운 개념을 바탕으로 **직접 동작하는 결과물**을 만듭니다.
- “코드를 읽는 학습”보다 **작동하는 경험**을 우선합니다.
- 목적: **브라우저에서 움직이는 JS 감각** 만들기

✅ 주로 하는 것

- DOM 조작/이벤트 기반 기능 만들기
- 상태(state) 흉내내기 / 렌더링 흐름 이해
- 비동기 통신(fetch)로 데이터 붙이기
- 작은 컴포넌트/모듈 단위 실습

📂 바로가기: [js-core](./js-core)

---

## 🧭 추천 학습 흐름 (효율 루트)

1. `js-beginner`에서 개념을 “한 번” 잡는다
2. 바로 `js-core`에서 **같은 주제를 작은 기능으로 만든다**
3. 만들면서 막힌 개념만 다시 `js-beginner`로 돌아가 보완한다

> ✅ “이해 → 구현 → 다시 이해” 사이클로 학습합니다.

---

## ✅ 학습 규칙 (내 기준)

- 주석은 **“왜(WHY)” 중심**
- 예제는 **작고 직관적으로**
- 같은 주제라도
   - `js-beginner` = 설명/정리
   - `js-core` = 동작하는 결과물
     로 분리합니다.
